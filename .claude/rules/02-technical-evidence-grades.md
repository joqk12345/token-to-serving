# Technical Evidence Grades

Grade every important technical claim before it becomes chapter prose.

## A — Strong Technical Anchor

Use as factual anchor:

- course lecture PDF;
- original research paper;
- official documentation;
- source code;
- reproducible benchmark artifact;
- standards or API documentation;
- paper artifact with explicit model/hardware/workload setup.

## B — Credible Secondary Support

Usable with attribution:

- high-quality engineering blog;
- author talk or conference presentation;
- reputable technical analysis;
- benchmark report with partial but meaningful setup detail;
- widely accepted textbook explanation.

## C — Weak or Context-Dependent

Do not use alone for chapter-level claims:

- community tutorial;
- issue thread;
- unreproduced performance claim;
- ambiguous benchmark;
- summary that omits hardware/workload conditions.

## D — Not a Factual Anchor

Do not use as support:

- viral claim;
- marketing number without setup;
- unsourced chart;
- vague "people say" consensus;
- benchmark screenshot with no reproducible context.

## Rule

A chapter-level claim cannot rest only on C-grade evidence. Performance claims require conditions. If conditions are unknown, say so.
