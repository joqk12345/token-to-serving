## Core Terms

- `decode` ‚Äî The autoregressive inference phase that generates new tokens, typically one step at a time per sequence. First defined in [Chapter 12](chapters/12-inference-cost-model.md).
- `KV cache` ‚Äî Cached key and value tensors from prior attention steps, reused during autoregressive decoding. First defined in [Chapter 12](chapters/12-inference-cost-model.md).
- `LLM systems` ‚Äî The full stack required to train, adapt, optimize, and serve large language models: model objective, architecture, kernels, frameworks, distributed training, inference runtime, and serving infrastructure. First defined in [Chapter 1](chapters/01-why-llm-systems.md).
- `model-algorithm-system co-design` ‚Äî Joint design across model architecture, training/inference algorithms, software runtime, and hardware constraints. First defined in [Chapter 1](chapters/01-why-llm-systems.md).
- `prefill` ‚Äî The inference phase that processes the prompt/context and builds the initial KV cache. First defined in [Chapter 12](chapters/12-inference-cost-model.md).
- `token` ‚Äî A discrete model input/output unit produced by a tokenizer; not necessarily a word. First defined in [Chapter 2](chapters/02-tokens-probability-transformers.md).
