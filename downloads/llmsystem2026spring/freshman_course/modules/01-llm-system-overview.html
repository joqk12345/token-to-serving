这是一篇面向大学新生的导学文档，根据课程课件重新组织。原始 PDF 仍保留在文末，供深入阅读。

学习目标

  • 区分模型、系统和应用
  • 理解为什么 LLM 需要专门的系统支持
  • 建立后续课程的知识地图

核心解释

LLM 的核心任务可以简化为“根据上下文预测下一个 token”。系统课程关心的是怎样让这个预测过程在真实机器上可训练、可部署、可扩展。

一个完整 LLM 系统通常包括数据处理、模型结构、训练框架、硬件加速、推理服务、监控和评测。任何一环太慢或太贵，都会影响最终用户体验。

新生不需要一开始记住所有算法名称，先抓住三条线：模型怎样计算，硬件怎样执行，服务怎样面对很多用户请求。

课堂例子

把 LLM 想成校园问答助手。模型负责生成回答，GPU 负责快速计算，服务系统负责同时处理很多同学的请求，并尽量让每个人少等。

检查问题

  1. 如果模型参数不变，系统层面还能优化什么？
  2. 为什么“能跑一次”和“能服务很多用户”不是一回事？
