从 Token 到 Transformer

Tokenization 与 Embedding

这是一篇面向大学新生的导学文档，根据课程课件重新组织。原始 PDF 仍保留在文末，供深入阅读。

学习目标

  • 理解 token 是模型输入单位
  • 认识 BPE 等分词方法的动机
  • 理解 embedding 的作用

核心解释

计算机不能直接处理自然语言文本，必须先把文本拆成 token 并映射成编号。

tokenization 需要在词表大小、未知词处理、多语言支持和压缩效率之间取舍。

embedding 把离散编号变成连续向量，后续 Transformer 层才能对它们做矩阵计算。

课堂例子

“internationalization” 可以拆成多个子词。这样模型不用为每个罕见长词都单独准备一个完整词表项。

检查问题

  1. token 太细或太粗会分别带来什么问题？
  2. 为什么需要 embedding，而不是直接用编号计算？
