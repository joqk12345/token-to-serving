Pallas Kernel 与 Splash Attention

这是一篇面向大学新生的导学文档，根据课程课件重新组织。原始 PDF 仍保留在文末，供深入阅读。

学习目标

  • 理解自定义 kernel 的动机
  • 认识 tile/block 思想
  • 知道 attention 优化要贴近硬件

核心解释

通用框架不一定能为每个新模型结构生成最优代码，自定义 kernel 允许开发者控制更底层的数据搬运和并行方式。

tile 思想把大矩阵切成小块，让片上内存可以反复复用局部数据。

Splash Attention 等方法关注如何在特定硬件上更有效地执行 attention。

课堂例子

如果普通公交路线绕路，定制班车可以更快到达。自定义 kernel 就是在重要路线旁边修一条更直接的路。

检查问题

  1. 什么时候值得写自定义 kernel？
  2. tile 为什么有助于内存复用？
