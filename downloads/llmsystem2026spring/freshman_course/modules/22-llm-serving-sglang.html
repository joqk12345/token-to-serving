LLM Serving 与调度

这是一篇面向大学新生的导学文档，根据课程课件重新组织。原始 PDF 仍保留在文末，供深入阅读。

学习目标

  • 理解在线推理服务的目标
  • 认识请求调度和缓存复用
  • 区分延迟和吞吐

核心解释

LLM serving 面对的是持续到来的用户请求，系统既要单个请求快，也要整体吞吐高。

请求长度、生成长度和到达时间都不同，调度器需要决定哪些请求一起执行。

缓存复用可以减少重复计算，尤其在多个请求共享前缀或上下文时。

课堂例子

奶茶店既要让单个顾客少等，也要让整队顾客流动快。批处理和调度就是在这两个目标之间平衡。

检查问题

  1. 延迟和吞吐有什么区别？
  2. 为什么相同前缀可以帮助缓存复用？
