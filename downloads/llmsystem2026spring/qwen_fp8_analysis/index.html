R1-R8 怎么读

R 表示 profile ranking bucket，也就是按耗时占比排出来的开销类别编号；它不是 Transformer layer 编号， 也不是执行顺序编号。R1 只是当前 profile 中最大的 bucket，R2 是第二大的 bucket，以此类推。

FP8 理论收益空间

244GB → 122GB122B 权重从 BF16 到 FP8，理论权重显存约减半，scale/metadata 另计。
20GB → 10GBA10B active 参数每 token 权重读取量约减半，decode memory-bound 时最明显。
1.4x ~ 1.8x更现实的端到端收益区间，需 FP8 GEMM、Fused QKV、FP8 KV cache 都生效。
2x+局部 kernel 上限，通常需要 Hopper/Blackwell FP8 Tensor Core、FA3/FA4 和调度配合。

FP8 报告给出的工程经验：Opt-0 全流程 FP8 仅大 batch 最多 19%，小 batch 可负优化； CUTLASS FP8 GEMM 平均 1.33x，Fused QKV 平均 1.38x，加入 FP8 KV cache 后平均 1.54x。 因此这张图强调“FP8 路径 + kernel 融合 + KV cache”必须同时成立。

图 2：dense GEMM / DG + NvJet 是什么

这里的 dense GEMM 指非 MoE 的普通 dense linear 矩阵乘法，主要来自 QKV projection、O projection、部分输出/辅助 linear。 `DG + NvJet` 按 profile label 理解为 dense GEMM 进入的优化 kernel/backend 路径，不是 MoE expert GEMM。

图 3：TP8 Transformer Block 内通信

TP8 表示一个 Transformer block 的大矩阵按 tensor parallel 切到 8 张 GPU。每张 GPU 计算 partial output， 在某些 projection 或 MLP/MoE 边界需要 all_reduce / reduce_scatter / all_gather 同步。

图 4：TP8 跨机器部署通信

如果 TP8 跨机器，all_reduce 不再只走机内 NVLink/NVSwitch，还会跨节点网络。跨机器 TP 会显著放大 R2， 因为每个 block 的同步都可能穿过较慢链路。

图 5：不同 layer 之间的关系

Hybrid Attention 往往不是每层完全相同。某些层承担 full/global 信息汇聚，某些层使用 sliding/local attention； 每个 layer 基本对应一个 Transformer block：Attention 子层 + MoE/MLP 子层 + residual/norm。 TP8 通信可能在每个 block 的 dense projection 或 MoE/MLP 边界出现。

图 6：Layer 与 Transformer Block 的关系

在 decoder-only LLM 中，一个 layer 通常就是一个重复的 Transformer block。每个 block 内部先走 attention 子层， 再走 MLP/MoE 子层，两个子层周围都有 RMSNorm 和 residual。Hybrid Attention 表示不同 block 的 attention 子层可以采用不同模式。

图 7：MoE / MLP 边界上的 TP8 同步

这张图是把图 1 的 MoE Feed Forward Layer 展开：图 1 里的 Router/Gate、Top-k Expert up GEMM、 Expert gate GEMM、SiLU/乘法、Expert down GEMM 在这里被串成执行路径；绿色节点是这些模块边界上可能发生的 TP8/EP collectives。

图 8：TP8 层间传递与一机 8 卡 MoE Expert 分布动态图

这张动态图把两个概念放在一起：上半部分表示 TP8 下每层输出通常以 8 个 shard 在 GPU0-GPU7 之间传递， 并在必要边界触发 all_reduce / reduce_scatter / all_gather；下半部分表示一机 8 卡时，MoE experts 按 owner GPU 分布， router 选择 top-k experts 后把 token dispatch 到对应 GPU，本地执行 grouped expert GEMM，再 combine 回下一层输入。
